> Hand-written (no Python/`memlog.py` in this environment). Append-only audit trail of build decisions.

- **event** — Build started. Intent: convert + generalize an existing project-specific skill into a reusable, project-agnostic autonomous improvement workflow. Source: `conformloc/.claude/commands/improve.md` + `improve.js`. Driven autonomously by the agent as the user's delegated proxy ("fais tout le nécessaire", report at end).
- **decision** — Name `continuous-improve` (standalone, no `bmad-` prefix — user creation, not official BMad). Output folder: `{project-root}/skills` per `bmb/config.yaml`.
- **decision** — No scripts shipped. The host environment has no Python; the work is judgment-heavy (discovery, review, capitalization); deterministic bits the model does reliably. Keeps the skill fully portable across arbitrary repos. Consequence: built skill does NOT use the runtime memlog pattern — its cross-run memory is git + the backlog file. This build memlog is hand-written.
- **decision** — Working-state strategy = **structured working artifact** (git-committed backlog at `{workflow.backlog_path}`) + git branch state. Not a memlog: the skill runs autonomously, single-session per run, and state must survive as committed files, not conversation.
- **decision** — Persona = **expert operator** (autonomous, unsupervised), NOT the BMad facilitator persona — per producing-workflow-patterns, the facilitator stance is dropped for expert-operator utilities.
- **decision** — Generalization: stripped all ConformLoc/LegalTech specifics (8 dimensions, RGPD domains, Turborepo/pnpm, fixed tracking file). Replaced with **runtime discovery** in the Plan stage (read CLAUDE.md/docs, infer verify command, bootstrap backlog). "Verify legal claims on official source" generalized to "verify any factual/external claim before asserting."
- **decision** — Preserved transferable architecture: Setup→Plan→Execute→Review→Ship; independent **review subagent** that can block + bounded 3-round correction loop + revert; rolling preprod→base PR; prompt + nested-CLAUDE.md capitalization. Base branch auto-detected (not hardcoded `main`).
- **decision** — Emitted customize.toml (answer to The Ask = YES). Justification: the skill runs across many repos with different branch names / verify commands; per-project pins genuinely earn their place. Surface = universal four + scalars (work_branch, base_branch, backlog_path, verify_command) + one append-only `finalize_reviewers` standards array. NO boolean toggles — "never auto-merge" kept as a hard SKILL.md rule, not a switch.
- **decision** — Carved two references (git-flow.md, review-gate.md) and one asset (backlog-template.md) to keep SKILL.md within token budget; each standalone; gotchas (never-commit-base, never-auto-merge, force-with-lease, atomic, no-complaisance) kept inline in SKILL.md per the "gotchas don't carve" rule.
- **decision** — Intents: `run` (default cycle) + lightweight read-only `status`. First `run` bootstraps the backlog (graceful degradation), so no separate `init` intent.
