## Pitch en 10 secondes
MaintienWeb : surveille, met a jour et securise ton site WordPress automatiquement pour 29 EUR/mois. Arrete de payer 300 EUR a ton webmaster pour cliquer sur "Mettre a jour".

---

## La Cible

**Qui paye** : Dirigeant de TPE/PME (1-20 salaries) qui possede un site WordPress et paye actuellement un freelance ou une agence pour la maintenance.

**Profil type** :
- TPE avec un site vitrine ou e-commerce WordPress
- Paye 100-500 EUR/mois a un webmaster/agence pour : mises a jour WordPress + plugins, sauvegardes, monitoring uptime, petits correctifs, rapport mensuel
- Ne comprend rien au technique, veut juste "que ca marche"
- A deja eu un site pirate ou casse apres une mise a jour ratee
- Budget digital existant (hebergeur, domaine, maintenance)

**Taille du marche France** :
- 43% des sites web dans le monde tournent sous WordPress (source W3Techs 2025)
- En France, estimation de 2 a 3 millions de sites WordPress actifs
- Cible : TPE/PME avec 1-20 salaries qui payent deja un prestataire maintenance = estimation **150 000 a 300 000 cibles**
- Largement au-dessus du seuil de 500

**Decideur unique** : OUI. Le dirigeant TPE ou le responsable marketing digital decide seul. Pas de comite, pas de DSI.

**Budget deja alloue** : OUI. Ces TPE payent DEJA un prestataire 100-500 EUR/mois. Le budget existe. On le remplace par 29-49 EUR/mois.

---

## La Douleur

**En 1 phrase** : Les dirigeants de TPE payent 200-500 EUR/mois a un freelance/agence pour une maintenance WordPress qui consiste essentiellement a cliquer sur "mettre a jour" et faire une sauvegarde -- une tache 100% automatisable.

**Detail de la douleur** :
1. **Surpaiement massif** : Le freelance facture 200-500 EUR/mois pour 30 minutes de travail reel (mises a jour + sauvegarde + coup d'oeil). Le ratio valeur/prix est catastrophique.
2. **Manque de reactivite** : Quand le site tombe, le freelance repond en 24-48h. Le dirigeant perd des clients en attendant.
3. **Opacite** : Le dirigeant ne sait pas ce que le prestataire fait vraiment. Le "rapport mensuel" est souvent un PDF generic.
4. **Dependance** : Si le freelance arrete, disparait ou augmente ses prix, le dirigeant est bloque.
5. **Peur** : Le site est un canal commercial crucial. La peur du piratage, de la perte de donnees ou du crash post-mise-a-jour est reelle.

---

## Le Declencheur d'Achat

**Evenement qui force l'action** :
- Le site tombe en panne et le freelance met 48h a repondre
- La facture de maintenance augmente sans explication
- Le dirigeant decouvre que son site a ete pirate
- Un confrere lui montre un outil qui fait ca automatiquement pour 29 EUR/mois

**Autres declencheurs** :
- Reception de la facture annuelle de maintenance (effet "rappel du cout")
- Recherche Google "maintenance WordPress pas cher" ou "comment securiser mon site WordPress"
- Post LinkedIn/Facebook sur les arnaques de maintenance web

### Analyse concurrentielle
- **ManageWP (GoDaddy) est un acteur a 100M$+** : GoDaddy capitalise a plus de 20 milliards USD. MAIS ManageWP est un outil pour AGENCES/DEVS, pas pour dirigeants TPE. L'interface est technique, en anglais, et presuppose des connaissances WordPress. Le gap est dans le POSITIONNEMENT (TPE non-tech, francophone, self-service) pas dans la technologie.
- **WP Umbrella est francais** mais cible explicitement les freelances et agences WordPress, pas les dirigeants TPE. Leur landing page dit "for WordPress professionals".
- Le creneau "maintenance WordPress cle-en-main pour le dirigeant non-tech" est mal adresse par les outils existants.

## La Differentiation

### Positionnement : "Ton webmaster automatique a 29 EUR/mois"

1. **Zero jargon technique** : L'interface ne parle pas de "plugins", "core updates" ou "cron jobs". Elle dit "Votre site est a jour et securise" ou "Attention : 2 actions requises".
2. **Dashboard pour non-technicien** : Score de sante du site (vert/orange/rouge), historique des actions, alertes en francais clair.
3. **Mises a jour intelligentes** : Mise a jour automatique avec snapshot pre-update. Si le site casse, rollback automatique en 1 clic. Pas besoin de savoir ce qu'est un "rollback".
4. **Rapport mensuel auto-genere** : PDF brande avec le logo de l'entreprise, listant toutes les actions effectuees. Le dirigeant peut le montrer a son comptable/associe.
5. **Monitoring + alertes** : Uptime monitoring, alertes SMS/email si le site tombe, scan de securite hebdomadaire.
6. **Support en francais** : Chat + email en francais, pas un forum US.

### Ce que MaintienWeb n'est PAS
- Pas un hebergeur (complementaire)
- Pas une agence web (pas de creation/modification de contenu)
- Pas un outil pour developpeurs (pas de terminal, pas d'API)
- Pas un responsable de la securite (outil de monitoring, pas de garantie de zero-breach)

## Faisabilite Technique (MVP)

### Stack
- **Frontend** : Next.js + React (competence Adrien)
- **Backend** : NestJS ou API routes Next.js
- **DB** : Supabase (PostgreSQL)
- **WordPress API** : Utilisation de l'API REST WordPress + WP-CLI pour les mises a jour a distance
- **Agent WordPress** : Plugin leger installe sur le site client qui communique avec le backend MaintienWeb
- **Monitoring** : Cron jobs Supabase Edge Functions pour uptime checks (ping toutes les 5 minutes)
- **PDF reports** : @react-pdf/renderer
- **Hosting** : Vercel
- **Paiement** : Stripe

### MVP en 8-10 semaines
1. **Semaine 1-2** : Plugin WordPress agent (ping + inventaire plugins/themes/core version)
2. **Semaine 3-4** : Backend : collecte des donnees agent, detection des mises a jour disponibles
3. **Semaine 5-6** : Mise a jour automatique via WP REST API/WP-CLI + snapshot pre-update + rollback
4. **Semaine 7-8** : Dashboard sante du site + alertes email/SMS + monitoring uptime
5. **Semaine 9-10** : Rapport PDF mensuel + Stripe + landing page + scan gratuit (lead magnet)

### Complexite technique
- **ATTENTION** : L'interaction avec des sites WordPress distants est techniquement non-triviale. Necessite un plugin agent installe cote client. La diversite des configurations WordPress (hebergeurs, versions PHP, plugins en conflit) rend les mises a jour risquees.
- Mitigation : commencer avec les mises a jour mineures uniquement + snapshot systematique + rollback automatique si erreur HTTP 500 detectee post-update.

### Cout infrastructure
- Supabase Pro : 25 EUR/mois
- Vercel Pro : 20 EUR/mois
- Uptime monitoring (externe ou Supabase cron) : 0-10 EUR/mois
- **Total : ~50-55 EUR/mois** -- dans le budget

## Verdict

### RETENUE -- Priorite moyenne

**Pourquoi cette idee est interessante** :
1. Budget deja alloue : les TPE payent DEJA 100-500 EUR/mois pour ce service
2. Ratio prix/valeur imbattable : 29 EUR vs 300 EUR pour le meme resultat
3. Scan gratuit = lead magnet puissant pour l'acquisition
4. Marche enorme (WordPress = 43% du web)
5. Excellent projet CV : plugin WordPress, APIs distantes, monitoring, PDF generation, SaaS B2B

**Pourquoi en priorite moyenne (et pas haute)** :
1. Douleur latente, pas aigue -- le dirigeant doit avoir un declencheur
2. GoDaddy/ManageWP est un geant qui pourrait pivoter (meme si c'est peu probable)
3. Complexite technique superieure aux autres idees (agent WordPress, configs heterogenes)
4. Support potentiellement lourd si les mises a jour automatiques causent des problemes
5. Le dirigeant TPE peut avoir du mal a installer le plugin agent tout seul (friction onboarding)

**Recommandation** : Idee viable mais moins prioritaire que Qualiopi Pilot (angle 5, 18/20) ou Relance Express (angle 6, 17/20). A considerer si les idees prioritaires sont deja lancees ou si Adrien souhaite se positionner dans la niche WordPress/web.
