# Top 10 Métiers Adjacents au Développement Web

**Contexte :** Métiers permettant d'intégrer une équipe IT, avec potentiel d'évolution vers développeur web en interne.

---

## 1. QA Engineer / Testeur automatisation

### C'est quoi ?
Le QA (Quality Assurance) Engineer est le gardien de la qualité logicielle. Il conçoit et exécute des tests pour s'assurer que les applications fonctionnent correctement avant leur mise en production. Le testeur automatisation va plus loin : il écrit du code (scripts) pour automatiser ces tests au lieu de les faire à la main. Concrètement, tu simules des parcours utilisateur (cliquer ici, remplir un formulaire, vérifier que la page affiche le bon résultat) via du code.

### Au quotidien
- Écrire des scénarios de test automatisés (Cypress, Playwright, Selenium)
- Identifier et reporter les bugs avec précision
- Travailler avec les devs pour comprendre les fonctionnalités
- Mettre en place des pipelines de tests dans la CI/CD
- Faire de la revue de qualité sur les pull requests

### Tes compétences actuelles vs. requises

| Requis | Tu l'as ? | Détail |
|--------|-----------|--------|
| JavaScript/TypeScript | **Oui** | C'est ta stack principale |
| Logique de test | **Oui** | Tu développes déjà, tu comprends les cas limites |
| Connaissance web (HTML, DOM, API) | **Oui** | Tu fais du React/Next.js |
| Outils de test (Cypress, Playwright) | **Partiel** | Tu connais probablement Jest, mais les outils E2E spécifiques nécessitent une prise en main |
| CI/CD | **Oui** | Tu utilises GitHub Actions, Vercel |
| Rigueur / Attention au détail | **Oui** | Force identifiée dans ton profil |

### Formation nécessaire
**Faible.** 1-2 semaines d'autoformation suffisent :
- Apprendre Playwright ou Cypress (tutoriels officiels, très bien documentés)
- Comprendre les patterns de test (Page Object Model, fixtures)
- Faire 1-2 projets démo sur GitHub pour montrer la compétence

### Facilité d'embauche
**Élevée.** Les QA qui savent coder sont rares. La plupart des testeurs viennent de parcours non-techniques. Toi, tu arrives avec TypeScript + compréhension full stack = profil premium pour ce poste. Beaucoup d'offres en Île-de-France, en CDI, y compris en ESN.

### Potentiel de transition vers dev
**Très fort.** Tu es dans l'équipe dev, tu lis le code source, tu comprends l'architecture. Beaucoup de QA Engineers deviennent devs en 6-12 mois en interne.

## 2. Intégrateur Web / Développeur front junior

### C'est quoi ?
L'intégrateur web transforme les maquettes graphiques (Figma, Adobe XD) en pages web fonctionnelles. Il se concentre sur le HTML, CSS et JavaScript côté navigateur. C'est la version "spécialisée front" du développeur web. Le titre est moins intimidant pour les recruteurs que "développeur full stack" et les attentes techniques sont plus ciblées.

### Au quotidien
- Transformer des maquettes Figma en composants HTML/CSS/JS
- Assurer le responsive design (mobile, tablette, desktop)
- Respecter les normes d'accessibilité (WCAG)
- Optimiser les performances front (images, fonts, animations)
- Collaborer avec les designers et les développeurs back-end

### Tes compétences actuelles vs. requises

| Requis | Tu l'as ? | Détail |
|--------|-----------|--------|
| HTML/CSS | **Oui** | Base de ta formation et tes projets |
| JavaScript | **Oui** | Maîtrisé |
| React | **Oui** | C'est ta techno principale |
| Tailwind CSS | **Oui** | Dans ta stack |
| Figma (lecture) | **Partiel** | Tu sais probablement lire une maquette, mais l'aisance avec l'outil peut varier |
| Accessibilité (a11y) | **Partiel** | Connaissances de base probables, mais pas approfondi |
| Responsive design | **Oui** | Tes projets en production le démontrent |

### Formation nécessaire
**Très faible.** Tu es déjà surqualifié pour ce poste. Éventuellement :
- Rafraîchir les bases d'accessibilité web (ARIA, WCAG 2.1)
- Pratiquer l'intégration pixel-perfect depuis Figma

### Facilité d'embauche
**Correcte.** Les agences web recrutent régulièrement des intégrateurs. C'est moins en tension que QA automatisation, mais les offres existent. Attention : les salaires sont souvent plus bas (car perçu comme "moins technique"). En ESN ou agence, c'est un point d'entrée classique.

### Potentiel de transition vers dev
**Immédiat.** C'est déjà du développement front-end. La transition vers "développeur React" est naturelle en montrant tes capacités sur des sujets plus complexes (state management, API, architecture).

## 3. DevOps / Ops junior

### C'est quoi ?
Le DevOps fait le pont entre le développement et l'infrastructure. Il met en place et maintient les systèmes qui permettent de déployer, surveiller et faire tourner les applications. Concrètement : pipelines CI/CD, conteneurs Docker, serveurs, monitoring, automatisation. C'est le métier qui fait que le code des développeurs arrive en production de manière fiable.

### Au quotidien
- Configurer et maintenir des pipelines CI/CD (GitHub Actions, GitLab CI, Jenkins)
- Gérer des conteneurs Docker et orchestrateurs (Kubernetes)
- Administrer des serveurs cloud (AWS, GCP, Azure)
- Mettre en place du monitoring et des alertes
- Automatiser les tâches répétitives (scripts Bash, Python)
- Gérer la sécurité des infrastructures

### Tes compétences actuelles vs. requises

| Requis | Tu l'as ? | Détail |
|--------|-----------|--------|
| Docker | **Oui** | Utilisé dans Amazon Monitor |
| Git/GitHub | **Oui** | Quotidien |
| CI/CD basique | **Oui** | Vercel, GitHub Actions |
| Linux / ligne de commande | **Partiel** | Tu utilises un terminal mais l'administration Linux avancée est un autre niveau |
| Cloud (AWS/GCP/Azure) | **Non** | Tu déploies sur Vercel, pas sur de l'infra cloud brute |
| Kubernetes | **Non** | Orchestration de conteneurs, c'est un monde à part |
| Scripting (Bash, Python) | **Partiel** | Tu codes en JS/TS, mais Bash/Python pour l'ops c'est un autre usage |
| Réseau / Sécurité | **Non** | DNS, pare-feu, certificats SSL — notions à acquérir |

### Formation nécessaire
**Moyenne à importante.** C'est le métier qui demande le plus de nouvelles compétences dans cette liste :
- Linux administration (2-4 semaines)
- Un cloud provider : AWS est le plus demandé (certification AWS Cloud Practitioner = 1-2 mois)
- Kubernetes basics (2-3 semaines)
- Approfondir Docker (1 semaine)

### Facilité d'embauche
**Très élevée.** C'est un des métiers IT les plus en pénurie. Mais le gap de compétences est réel pour toi. Si tu investis 2-3 mois de formation, tu deviens très employable. Les salaires sont aussi plus élevés que la moyenne dev junior.

### Potentiel de transition vers dev
**Latéral.** Tu ne reviendrais pas naturellement vers du dev web classique. Tu évoluerais plutôt vers du DevOps senior, SRE (Site Reliability Engineer), ou Platform Engineer. C'est une bifurcation, pas un tremplin vers dev front/back.

## 4. Low-code / No-code Developer

### C'est quoi ?
Le développeur low-code/no-code construit des applications métier en utilisant des plateformes visuelles (Bubble, Webflow, Retool, Make/Zapier, n8n, PowerApps). Au lieu d'écrire tout le code à la main, il assemble des composants visuels et configure des workflows. C'est de la création d'applications sans (ou avec peu de) code traditionnel. Les entreprises adorent ça car c'est rapide et moins cher.

### Au quotidien
- Construire des applications internes (dashboards, formulaires, workflows)
- Automatiser des processus métier (quand X arrive, faire Y)
- Connecter des outils entre eux via des API (CRM + email + Slack + base de données)
- Former les équipes non-techniques à utiliser les outils
- Maintenir et faire évoluer les applications créées

### Tes compétences actuelles vs. requises

| Requis | Tu l'as ? | Détail |
|--------|-----------|--------|
| Logique de programmation | **Oui** | Surqualifié. Tu comprends les boucles, conditions, API |
| API / Webhooks | **Oui** | Tu intègres des API dans tes projets (FFBB, Stripe, Shopify) |
| Bases de données | **Oui** | PostgreSQL, MongoDB, Supabase |
| Plateformes no-code (Bubble, Webflow, Retool) | **Non** | Chaque plateforme a son propre écosystème à apprendre |
| Automatisation (Make, Zapier, n8n) | **Non** | Mais la logique est simple quand tu sais coder |
| Communication avec des non-techniques | **Partiel** | Tu sais expliquer, mais c'est un muscle à développer |

### Formation nécessaire
**Faible à moyenne.** La logique tu l'as, il faut juste apprendre les outils :
- Choisir 1-2 plateformes et les maîtriser (Bubble + Make, ou Retool + n8n) — 2-3 semaines chacune
- Faire des projets démo pour montrer ta maîtrise

### Facilité d'embauche
**En croissance.** Les PME et startups recrutent de plus en plus pour ça. Le titre exact varie : "Ops Developer", "Business Automation Specialist", "No-code Developer". Moins d'offres que QA ou DevOps, mais la concurrence est moindre.

### Potentiel de transition vers dev
**Bon.** Quand les limites du no-code apparaissent (et elles apparaissent toujours), tu es la personne idéale pour dire "je peux coder ça en custom". C'est un tremplin naturel.

## 5. Support Technique N2/N3 (SaaS)

### C'est quoi ?
Le support technique niveau 2 ou 3, c'est l'escalade après le support client classique (N1). Quand un problème dépasse les FAQ et les réponses standard, ça arrive chez toi. Tu diagnostiques des bugs, tu analyses des logs, tu reproduis des problèmes techniques, et tu fais le lien entre le client et l'équipe dev. En SaaS, tu travailles sur un produit web, donc tu es en terrain connu.

### Au quotidien
- Diagnostiquer des problèmes techniques remontés par les clients
- Analyser des logs, requêtes API, bases de données pour trouver la cause
- Reproduire les bugs et documenter pour l'équipe dev
- Répondre aux clients avec des explications techniques claires
- Remonter les patterns de bugs récurrents à l'équipe produit
- Écrire de la documentation technique

### Tes compétences actuelles vs. requises

| Requis | Tu l'as ? | Détail |
|--------|-----------|--------|
| Compréhension web (HTTP, API, BDD) | **Oui** | Tu construis des apps web complètes |
| Lecture de logs / debugging | **Oui** | Tu débugges tes propres apps |
| SQL basique | **Oui** | PostgreSQL dans ta stack |
| Communication écrite claire | **Oui** | Rigueur analytique identifiée |
| Patience / Calme | **Oui** | Force majeure de ton profil |
| Gestion de la frustration client | **Oui** | Arbitrage national = gestion de conflits |
| Outils support (Zendesk, Intercom) | **Non** | Mais ce sont des outils simples à prendre en main |

### Formation nécessaire
**Quasi nulle.** Tu as déjà toutes les compétences. Juste se familiariser avec les outils de ticketing (Zendesk, Jira Service Management) en quelques jours.

### Facilité d'embauche
**Très élevée.** Les SaaS recrutent du support technique en permanence. C'est un des postes avec le moins de friction à l'embauche dans la tech. Ton profil technique te place immédiatement au-dessus de la moyenne des candidats.

### Potentiel de transition vers dev
**Classique et éprouvé.** C'est un chemin connu : support N3 → tu commences à fixer des bugs toi-même → on te propose un poste dev. Beaucoup de développeurs en poste aujourd'hui sont passés par le support. Tu connais le produit mieux que les devs.

**Attention :** Le risque c'est de rester "coincé" en support si l'entreprise n'a pas de culture de mobilité interne. Bien choisir la boîte (startups/scale-ups avec culture de promotion interne).

## 6. Analyste fonctionnel / Business Analyst IT

### C'est quoi ?
L'analyste fonctionnel fait le pont entre le métier (les besoins business) et la technique (l'équipe de développement). Il recueille les besoins des utilisateurs, les traduit en spécifications fonctionnelles, et s'assure que ce qui est développé correspond à ce qui est demandé. C'est le traducteur entre ceux qui veulent le produit et ceux qui le construisent.

### Au quotidien
- Animer des ateliers avec les utilisateurs pour comprendre leurs besoins
- Rédiger des spécifications fonctionnelles et user stories
- Créer des maquettes/wireframes pour illustrer les solutions
- Valider les développements par rapport aux spécifications
- Gérer le backlog avec le Product Owner
- Documenter les processus métier

### Tes compétences actuelles vs. requises

| Requis | Tu l'as ? | Détail |
|--------|-----------|--------|
| Compréhension technique | **Oui** | Tu es dev, tu comprends les contraintes |
| Analyse et structuration | **Oui** | Recherche exhaustive = ta compétence unique |
| Rédaction de specs | **Partiel** | Tu sais documenter du code, mais les specs fonctionnelles c'est un format spécifique |
| Animation d'ateliers | **Partiel** | Tu as l'expérience du management d'équipe, mais l'animation d'ateliers métier est différente |
| UML / Modélisation | **Non** | Diagrammes de cas d'usage, séquence, activité — à apprendre |
| Outils (Jira, Confluence, Miro) | **Partiel** | Tu connais probablement Jira, le reste s'apprend vite |

### Formation nécessaire
**Moyenne.** La méthodologie est spécifique :
- Apprendre les techniques de recueil de besoins (2 semaines)
- UML / modélisation basique (1-2 semaines)
- Se former aux user stories et critères d'acceptation (1 semaine)
- Optionnel : certification IIBA (International Institute of Business Analysis)

### Facilité d'embauche
**Élevée, surtout en ESN.** Les ESN (Capgemini, Sopra Steria, Accenture...) recrutent massivement des analystes fonctionnels. C'est souvent un premier poste accessible. Ton profil technique est un avantage car beaucoup d'analystes n'ont pas de background dev.

### Potentiel de transition vers dev
**Moyen.** Tu es côté spécifications, pas côté code au quotidien. La transition est possible mais demande de la proactivité : proposer de coder des prototypes, prendre des tâches de dev sur ton temps libre au boulot. C'est moins naturel qu'en QA ou support N3.

## 7. Chef de projet technique junior / Scrum Master

### C'est quoi ?
Le chef de projet technique coordonne la réalisation de projets informatiques : planning, suivi, coordination entre les équipes, gestion des risques. Le Scrum Master, lui, est le facilitateur d'une équipe Agile : il anime les cérémonies (daily, sprint review, rétrospective), élimine les obstacles, et protège l'équipe. Les deux rôles sont proches mais le Scrum Master est plus "servant leader" que chef.

### Au quotidien
- Planifier et suivre l'avancement des projets/sprints
- Animer les réunions d'équipe (daily stand-up, planning, rétro)
- Identifier et résoudre les blocages de l'équipe
- Communiquer l'état d'avancement aux parties prenantes
- Gérer les priorités et le backlog avec le Product Owner
- Garantir le respect des méthodologies (Agile, Scrum)

### Tes compétences actuelles vs. requises

| Requis | Tu l'as ? | Détail |
|--------|-----------|--------|
| Leadership | **Oui** | 10 ans de direction de club |
| Organisation / Planification | **Oui** | Gestion événements, saisons sportives |
| Communication | **Oui** | Communication institutionnelle (fédérations, mairies) |
| Méthodologie Agile/Scrum | **Partiel** | C'est dans ta stack méthodologique, mais la pratique en contexte IT est différente |
| Gestion de conflits | **Oui** | Arbitrage + éducation |
| Compréhension technique | **Oui** | Tu es dev, tu comprends ce que fait l'équipe |
| Outils (Jira, Trello, Notion) | **Partiel** | À approfondir dans un contexte pro IT |

### Formation nécessaire
**Moyenne :**
- Certification PSM I (Professional Scrum Master) — 2-4 semaines de préparation, examen en ligne (~150€)
- Cette certification ouvre beaucoup de portes et est reconnue partout
- Se former aux outils Jira/Confluence en contexte Scrum (1 semaine)

### Facilité d'embauche
**Correcte.** Les ESN recrutent des Scrum Masters, souvent après certification. Le marché est moins en tension que pour QA ou DevOps. Ton profil atypique (sport + management associatif) peut plaire à certains recruteurs qui veulent un Scrum Master humain, pas juste technique.

### Potentiel de transition vers dev
**Faible.** C'est le rôle qui t'éloigne le plus du code dans cette liste. Tu manages l'équipe mais tu ne codes pas. La transition vers dev nécessiterait un vrai changement de poste, pas une évolution naturelle. En revanche, tu pourrais évoluer vers Product Owner, Engineering Manager, ou CTO en startup.

## 8. Spécialiste IA / Prompt Engineer

### C'est quoi ?
Le spécialiste IA en entreprise aide les équipes à intégrer l'intelligence artificielle dans leurs processus et produits. Le Prompt Engineer conçoit des prompts optimisés pour les LLM (ChatGPT, Claude...) et met en place des workflows IA. C'est un rôle nouveau qui combine compréhension technique, créativité, et connaissance des outils IA. C'est exactement à l'intersection de tes compétences actuelles.

### Au quotidien
- Concevoir et optimiser des prompts pour des cas d'usage business
- Intégrer des API d'IA (OpenAI, Anthropic) dans des applications
- Mettre en place des pipelines RAG (Retrieval Augmented Generation)
- Former les équipes à l'utilisation efficace de l'IA
- Évaluer et benchmarker les modèles pour chaque cas d'usage
- Automatiser des workflows avec des agents IA

### Tes compétences actuelles vs. requises

| Requis | Tu l'as ? | Détail |
|--------|-----------|--------|
| Utilisation avancée des LLM | **Oui** | Claude Code, MCP, veille constante |
| Programmation (Python ou JS/TS) | **Oui** | TypeScript maîtrisé |
| Intégration API | **Oui** | Tu intègres des API dans tous tes projets |
| Prompt engineering | **Oui** | Tu le pratiques quotidiennement |
| RAG / Vector databases | **Partiel** | Concepts à approfondir si pas encore pratiqué |
| Fine-tuning / ML basics | **Non** | Pas nécessaire pour la plupart des postes, mais un plus |
| Évaluation et benchmarking | **Oui** | Ta compétence unique de recherche exhaustive |

### Formation nécessaire
**Faible.** Tu es déjà dans l'écosystème :
- Structurer tes connaissances IA (frameworks d'évaluation, patterns de prompts)
- Apprendre les bases RAG si pas fait (LangChain/LlamaIndex — 1-2 semaines)
- Constituer un portfolio de cas d'usage IA documentés

### Facilité d'embauche
**Moyenne, en croissance rapide.** Le marché est encore jeune. Les offres existent surtout dans les startups IA, les scale-ups tech-forward, et les grands groupes en transformation digitale. Moins d'offres que QA ou support, mais aussi beaucoup moins de candidats qualifiés. Le titre varie : "AI Engineer", "Prompt Engineer", "IA Specialist", "LLM Engineer".

### Potentiel de transition vers dev
**Excellent.** C'est le rôle qui te positionne le mieux pour le futur. Dev + IA c'est la direction du marché. Tu ne transitionnes pas "vers" dev, tu deviens un dev augmenté par l'IA, ce qui vaut plus qu'un dev classique.

## 9. Webmaster / Administrateur CMS

### C'est quoi ?
Le webmaster gère et maintient un ou plusieurs sites web existants. Il met à jour le contenu, assure la maintenance technique (mises à jour, sécurité, sauvegardes), résout les problèmes, et fait évoluer le site. L'administrateur CMS est spécialisé sur des plateformes comme WordPress, Drupal, ou Shopify. C'est un rôle moins "glamour" mais très concret et demandé.

### Au quotidien
- Mettre à jour le contenu des sites (textes, images, produits)
- Assurer la maintenance technique (mises à jour CMS, plugins, sécurité)
- Résoudre les bugs et problèmes d'affichage
- Optimiser le SEO technique
- Installer et configurer des plugins/extensions
- Gérer l'hébergement et les noms de domaine
- Former les équipes internes à l'utilisation du CMS

### Tes compétences actuelles vs. requises

| Requis | Tu l'as ? | Détail |
|--------|-----------|--------|
| HTML/CSS/JavaScript | **Oui** | Surqualifié |
| WordPress / CMS | **Partiel** | Tu connais le web mais chaque CMS a ses spécificités (thèmes, hooks WordPress, PHP basics) |
| Shopify | **Oui** | Projet Ekiboost |
| SEO basique | **Partiel** | Tu sais faire du Next.js SSR (bon pour le SEO), mais les techniques SEO spécifiques (meta, schema.org, search console) sont à approfondir |
| Hébergement / DNS | **Partiel** | Tu déploies sur Vercel, mais la gestion d'hébergement mutualisé/VPS c'est différent |
| Sécurité web basique | **Partiel** | Concepts connus, pratique spécifique CMS à acquérir |

### Formation nécessaire
**Faible :**
- WordPress : bases PHP + écosystème WordPress (1-2 semaines)
- SEO : bases techniques et outils (Google Search Console, Screaming Frog) — 1 semaine
- Optionnel : si tu vises Shopify, tu es déjà prêt

### Facilité d'embauche
**Élevée.** PME, associations, collectivités, agences — tout le monde a besoin de webmasters. Le niveau d'exigence est plus bas que pour un dev. Tu es surqualifié, ce qui te donne un avantage clair en entretien. Beaucoup d'offres en CDI, y compris hors Paris.

### Potentiel de transition vers dev
**Variable.** Ça dépend totalement de la structure :
- En agence web avec des devs → transition possible, tu montres que tu peux coder custom
- En PME où tu es seul sur le site → pas d'équipe dev, pas de transition
- **Choisir la bonne structure est crucial**

## 10. Customer Success Engineer (SaaS technique)

### C'est quoi ?
Le Customer Success Engineer (CSE) aide les clients d'un produit SaaS à tirer le maximum de valeur du produit. Contrairement au support (réactif — le client a un problème), le CSE est proactif : il accompagne, forme, configure, et s'assure que le client atteint ses objectifs. C'est un rôle technique mais orienté relation client. Ce n'est PAS de la vente : les clients sont déjà signés.

### Au quotidien
- Accompagner l'onboarding des nouveaux clients
- Configurer le produit selon les besoins spécifiques de chaque client
- Former les équipes clientes à l'utilisation du produit
- Identifier les opportunités d'upsell (sans vendre directement)
- Analyser les données d'usage pour anticiper les problèmes
- Faire le lien entre les retours clients et l'équipe produit/dev
- Créer de la documentation et des tutoriels

### Tes compétences actuelles vs. requises

| Requis | Tu l'as ? | Détail |
|--------|-----------|--------|
| Compréhension technique du produit | **Oui** | Tu comprends comment fonctionne un SaaS de l'intérieur |
| Communication claire | **Oui** | Formation + management associatif |
| Patience et pédagogie | **Oui** | Éducation nationale + formation jeunes entraîneurs |
| Analyse de données | **Partiel** | Tu sais lire une BDD, mais les outils analytics (Mixpanel, Amplitude) sont à découvrir |
| Anglais | **Oui** | Avancé — gros atout car beaucoup de SaaS sont internationaux |
| Vente / Upsell | **Non** | Mais ce n'est pas le coeur du poste, c'est secondaire |
| Outils CRM (HubSpot, Salesforce) | **Non** | À apprendre, mais c'est de la prise en main d'outil |

### Formation nécessaire
**Faible :**
- Se familiariser avec les outils CRM (HubSpot — gratuit pour apprendre) — 1 semaine
- Comprendre les métriques SaaS (churn, NRR, NPS) — quelques jours
- Optionnel : certification HubSpot (gratuite)

### Facilité d'embauche
**Élevée dans l'écosystème startup/scale-up.** Les SaaS B2B en croissance recrutent constamment des CSE. Ton profil technique est un avantage massif (la plupart des CSE ne savent pas coder). Beaucoup d'offres en Île-de-France, souvent avec de bonnes conditions (télétravail, avantages startup).

### Potentiel de transition vers dev
**Moyen.** Tu es côté client, pas côté code. Mais dans une petite structure (startup), les frontières sont floues et tu pourrais contribuer au produit. C'est aussi un bon tremplin vers Product Manager si tu veux évoluer différemment.

**Point d'attention pour toi :** Ce rôle demande une endurance relationnelle (appels clients réguliers). Avec ton profil d'introverti social et ton endurance sociale limitée, vérifie que la cadence te convient avant de t'engager.
