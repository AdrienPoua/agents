## Post (Copier-coller dans Buffer)

"Fais-moi un composant React."

Résultat : code générique inutilisable.

Voici ce qui marche vraiment :

❌ 𝗣𝗿𝗼𝗺𝗽𝘁 𝘃𝗮𝗴𝘂𝗲
→ "Crée un formulaire de contact"

✅ 𝗣𝗿𝗼𝗺𝗽𝘁 𝗽𝗿𝗲́𝗰𝗶𝘀
→ "Crée un formulaire de contact en React + TypeScript"
→ "Utilise react-hook-form + zod pour la validation"
→ "Champs : nom, email, message"
→ "Style avec Tailwind, design minimal"
→ "Gère les états loading/error/success"

La différence ?

Le premier me donne du code que je dois réécrire.

Le second me donne du code que je peux utiliser.

𝗠𝗼𝗻 𝘁𝗲𝗺𝗽𝗹𝗮𝘁𝗲 𝗱𝗲 𝗽𝗿𝗼𝗺𝗽𝘁 :

1. Contexte (stack, architecture)
2. Résultat attendu (user story)
3. Contraintes techniques
4. Exemple de code existant si pertinent

Plus tu donnes de contexte, meilleur est le résultat.

—

C'est quoi ton prompt le plus efficace ?
