## Critères OBLIGATOIRES (Non-négociables)

### 1. Pas de concurrent dominant
```
❌ ELIMINER si :
- Un acteur a levé 100M$+ (Datadog, Snyk, Notion...)
- Plus de 5 concurrents établis sur le marché
- Les gros (Google, Microsoft, Salesforce) sont déjà là
- Nécessite des millions en R&D pour être compétitif

✅ GARDER si :
- Niche verticale ignorée par les gros
- Solutions existantes inadaptées (trop chères, trop complexes, pas localisées)
- Marché émergent sans leader clair
```

### 2. Douleur ÉVIDENTE et URGENTE
```
La douleur doit être :
- Quantifiable (€ perdus, heures gaspillées, risque légal)
- Récurrente (pas un one-shot)
- Reconnue par la cible (ils savent qu'ils ont ce problème)

Questions à se poser :
- "Est-ce que la cible se réveille la nuit à cause de ce problème ?"
- "Est-ce qu'ils cherchent activement une solution ?"
- "Combien ça leur coûte de NE PAS résoudre ce problème ?"
```

### 3. Déclencheur d'achat CLAIR
```
Le meilleur déclencheur = un ÉVÉNEMENT qui force l'action :
- Deadline légale (obligation réglementaire)
- Audit imminent
- Sanction/amende reçue ou risquée
- Changement de situation (nouvelle responsabilité, croissance)
- Événement ponctuel (mariage, événement à organiser)

❌ MAUVAIS déclencheur : "Ce serait bien d'avoir..."
✅ BON déclencheur : "Je DOIS avoir ça avant le [date] sinon..."
```

### 4. Cible TROUVABLE facilement
```
CRITIQUE : Je dois pouvoir trouver mes prospects sur internet

Sources idéales :
- Base de données publique (data.gouv.fr, annuaires officiels)
- Liste d'entreprises par secteur (Societe.com, Pappers)
- Groupes LinkedIn/Facebook identifiables
- Événements/salons où ils se retrouvent
- Certifications/labels publics

❌ ELIMINER si :
- Cible diffuse, impossible à identifier
- Pas de canal d'acquisition clair
- Nécessite du networking "à l'ancienne"
```

### 5. Vente "évidente"
```
Le client doit comprendre la valeur en 10 secondes :
- "Tu as [PROBLÈME], je résous [PROBLÈME], ça coûte [PRIX]"
- ROI calculable facilement
- Pas besoin d'éduquer le marché

❌ ELIMINER si :
- Besoin d'expliquer pendant 30 min pourquoi c'est utile
- Le client ne sait pas qu'il a ce problème
- Vente complexe avec multiples décideurs
```

## Critères BONUS (Nice to have)

### Architecture technique impressionnante
```
Pour le CV, privilégier les projets avec :
- Event Sourcing / CQRS
- Multi-tenant architecture
- Real-time (WebSockets, SSE)
- Distributed systems
- ML/IA en production
- Intégrations API complexes
```

### Marché captif
```
Idéal : obligation légale ou certification obligatoire
- Le client N'A PAS LE CHOIX de s'équiper
- Rétention naturelle (coût de switching élevé)
```

### Pricing par usage ou événement
```
Plus facile à vendre que l'abonnement :
- "X€ par événement" (EventSafe)
- "X€ par lot" (SyndicFlow)
- "X€ par facture traitée"
```

## Critères ÉLIMINATOIRES (Red flags)

### Problème de l'œuf et la poule
```
❌ Marketplaces two-sided où il faut :
- Des vendeurs ET des acheteurs
- Des candidats ET des recruteurs
- Du contenu ET des lecteurs

Exception : si partenariat possible avec un acteur qui a déjà les deux côtés
```

### Complexité légale/réglementaire excessive
```
❌ ELIMINER si :
- Responsabilité légale directe (conseil juridique, médical)
- Certifications obligatoires coûteuses (HDS, agrément)
- Secteur ultra-réglementé (banque, santé lourde)

✅ OK si :
- Génération de templates/documents (pas de conseil)
- Outil d'aide à la conformité (pas de garantie)
- Disclaimer + CGU protègent
```

### Marché théorique sans preuves
```
❌ ELIMINER si :
- "Les amendes existent mais personne n'est sanctionné"
- "Le marché va exploser dans 5 ans"
- "Les gens devraient vouloir ça"

✅ GARDER si :
- Preuves de sanctions/contrôles réels
- Concurrents qui font du CA (validation marché)
- Témoignages de douleur sur forums/groupes
```

## Template d'Évaluation d'une Idée

```markdown
## [NOM DU PROJET]

### La Douleur
- **Qui souffre ?** [Cible précise]
- **De quoi ?** [Problème concret]
- **Combien ça coûte ?** [€ ou heures perdues]
- **Fréquence ?** [Quotidien/Hebdo/Annuel]

### Le Déclencheur d'Achat
- **Quel événement force l'action ?** [Deadline, audit, sanction...]
- **Quand ?** [Date précise si possible]
- **Quelle urgence ?** [Critique/Haute/Moyenne/Basse]

### L'Acquisition Client
- **Où trouver les prospects ?** [Base de données, groupe, salon...]
- **Combien sont-ils ?** [Taille du marché]
- **Comment les contacter ?** [Email, LinkedIn, téléphone...]
- **Quel message d'accroche ?** [1 phrase]

### La Concurrence
- **Qui existe ?** [Noms + pricing]
- **Pourquoi sont-ils inadaptés ?** [Trop cher, trop complexe, pas FR...]
- **Mon angle différenciant ?** [1 phrase]

### Technique
- **Complexité ?** [1-5]
- **Temps MVP ?** [Semaines]
- **Stack requise ?** [Technologies]
- **Impressionnant pour CV ?** [Oui/Non + pourquoi]

### Verdict
- **Score Douleur :** [1-5]
- **Score Déclencheur :** [1-5]
- **Score Acquisition :** [1-5]
- **Score Concurrence :** [1-5]
- **TOTAL :** [X/20]
- **Go/No-Go :** [Décision]
```

## Prompt pour Relancer une Recherche

```
Je cherche des idées de projets SaaS avec ces critères STRICTS :

1. **Douleur évidente** : Le client sait qu'il a ce problème et ça lui coûte cher (€ ou temps)

2. **Déclencheur d'achat clair** : Un événement force l'action (deadline légale, audit, sanction, obligation)

3. **Cible trouvable** : Je peux trouver les prospects via une base de données publique ou un canal identifiable (pas de networking)

4. **Pas de concurrent dominant** : Pas de géant avec 100M$+ levés, niche sous-servie ou solutions existantes inadaptées

5. **Vente évidente** : Le client comprend la valeur en 10 secondes, pas besoin d'éduquer le marché

6. **Pas de marketplace two-sided** : Pas de problème œuf/poule

7. **Complexité légale limitée** : Pas de responsabilité juridique directe, templates OK mais pas de conseil

Pour chaque idée, donne-moi :
- La douleur précise (qui, quoi, combien ça coûte)
- Le déclencheur d'achat (quel événement, quelle date)
- Comment trouver les clients (quelle base de données, quel canal)
- Les concurrents et pourquoi ils sont inadaptés
- La complexité technique et le temps MVP estimé
```

## Ressources Utiles

### Bases de données publiques France
- [data.gouv.fr](https://data.gouv.fr) - Données ouvertes françaises
- [Pappers](https://pappers.fr) - Info entreprises gratuites
- [Societe.com](https://societe.com) - Annuaire entreprises
- [INSEE SIRENE](https://sirene.fr) - Base établissements

### Groupes/Communautés par niche
- LinkedIn : groupes professionnels par métier
- Facebook : groupes "Formateurs indépendants", "Syndics bénévoles", etc.
- Forums spécialisés par industrie

### Validation marché
- Rechercher "[niche] + galère/problème/aide" sur Google
- Lire les avis négatifs des concurrents (ce qui manque)
- Poser des questions dans les groupes Facebook/LinkedIn
